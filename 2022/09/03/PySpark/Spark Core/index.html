Spark Core

Spark Core

Spark Core 是 Spark 平台的底层通用执行引擎，所有其他功能都建立在其之上。它提供了RDD和内存计算能力。

1. RDD简介

在分布式框架中，需要有一个统一的数据抽象对象，来实现分布式计算所需功能（分区控制、Shuffle、序列化、数据计算API等），这个抽象对象就是RDD——弹性分布式数据集。

RDD（Resilient Distributed Dataset），是Spark中最基本的数据抽象，代表一个不可变、可分区、里面的元素可并行计算的集合，所有的运算、操作都是建立在RDD数据结构的基础之上。

  • Resilient：RDD中数据可以存储在内存或磁盘中。
  • Distributed：RDD中的数据是分布式存储的，可用于分布式计算；
  • Dataset：一个数据集合，用户存放数据。

1.2 五大特性

  1. A list of paritions：RDD是有分区的；
  2. A function for computing each split：计算方法都有作用到每一个分区上；
  3. A list of dependencies on other RDDs：RDD之间是有相互依赖关系的
  4. a Partitioner for key-value RDDS：KV型RDD可以有分区
  5. a list of preferred location compute each split on：RDD分区数据读取尽量靠近数据所在地；

RDD分区

RDD的分区是RDD数据存储的最小单位。

RDD方法会作用在所有的分区上

依赖关系

KV型RDD可以有分区

默认分区器：Hash分区规则，也可以手动设置一个分区器

RDD分区规划，会尽量靠近数据所在的服务器

在初始RDD规划的时候，分区会尽量规划到存储数据所在的服务器上，因为这样可以走本地读取，避免网络读取。（确保并行计算能力的前提下，尽量确保本地读取。）

2. RDD编程

Spark RDD编程的程序入口对象是SparkContext对象。

两种创建方式：

  • 并行化集合创建，sc.parallelize(c[iterable], numSlices)

  • 读取外部数据源，通过sc.textFile(name, minPartitions)，默认分区数和文件大小有关

2.1 RDD算子

RDD算子（分布式集合对象上的API）分为两类：

  • Transformation算子：返回值仍然是RDD的，称为转换算子（map、reduceByKey）；
  • Action算子：返回值不是rdd的就是action算子（collect）；

2.1.1 Transformation算子

1、map算子：RDD数据map处理。

Return a new RDD by applying a function to each element of this RDD.

2、flatmap算子：先执行map操作，然后进行解除嵌套操作（就是flatten）。

Return a new RDD by first applying a function to all elements of this RDD, and then flattening the results.

3、reduceByKey算子：针对KV型RDD，自动按照Key分组，提供聚合逻辑，完成组内value的聚合操作。

Merge the values for each key using an associative and commutative reduce function.

4、mapValues算子：针对二元组RDD，对其内部的Value执行map操作。

Pass each value in the key-value pair RDD through a map function without changing the keys; this also retains the original RDD's partitioning.

5、filter算子：过滤想要的数据进行保存。

Return a new RDD containing only the elements that satisfy a predicate.

6、distinct算子：去重。

7、union算子：合并两个RDD并返回。

8、join算子：对两个RDD执行JOIN操作（内/外连接），只能用于二元元组。join()、leftouterJoin()。默认是对key进行分组。

9、intersection算子：两个RDD取交集。

10、glom算子：将RDD的数据加上嵌套，这个嵌套按照分区进行。主要为了看下哪些数据在一个分区

11、groupBy算子：对RDD数据进行分组。

Return an RDD of grouped items.

12、groupByKey算子：针对KV的RDD数据，按照Key进行分组。

13、reduceByKey算子：自动按照Key分组，然后按照提供的逻辑，完成组内数据的聚合。

  • reduceByKey的性能是远大于groupByKey+聚合逻辑的

14、sortBy算子：对RDD进行排序，按照你指定的排序依据，

sortBy(self, keyfunc, ascending=True, numPartitions=None)

15、sortByKey算子：对KV型进行排序，按照Key。

def sortByKey(self, ascending=True, numPartitions=None, keyfunc=lambda x: x):

2.1.2 Action算子

1、collect算子：将RDD各个分区的数据，统一收集到Driver中，形成一个List对象（数据量不能太大）。

Return a list that contains all of the elements in this RDD.。

2、first算子：取出RDD的第一个元素。

3、take算子：取出RDD的前N个元素。

4、top算子：对RDD数据集进行降序，并取前N个。

5、count算子：统计RDD数据数量。

6、countByKey算子：统计key出现过的次数，返回的是字典。

Count the number of elements for each key, and return the result to the master as a dictionary.

7、takeSample算子：随机抽样RDD数据。

def takeSample(self, withReplacement, num, seed=None):

8、takeOrdered算子：对RDD进行排序取前N个。

Get the N elements from an RDD ordered in ascending order or as specified by the optional key function.

9、foreach算子：对RDD每个元素，执行你提供的逻辑操作，无返回值。（跳过Driver，由分区所在的Executor直接执行）

Applies a function to all elements of this RDD.

10、saveAsTextFile算子：将RDD的数据写入文件中。（跳过Driver，由分区所在的Executor直接执行）

def saveAsTextFile(self, path, compressionCodecClass=None):

2、mapPartitions（Transformation）：已提到

3. RDD持久化

RDD的数据是过程数据，只在处理的过程中存在，一旦处理完成，就消失。

  • 这个特性可以最大化利用资源，从内存中清理旧RDD，给后续计算腾出内存空间

解决方法有两种：缓存与checkpoint。

3.1 RDD缓存

Spark提供了缓存API，通过调用API，将指定的RDD数据保留在内存或磁盘上。因为RDD是过程中的数据，旧的RDD会被清理。如果重新使用的话，需要从最开始计算一遍，浪费时间开销，所以有了缓存数据。
缓存级别：

特性：保留RDD之间的依赖关系。

缺点：缓存的数据在设计上是认为有丢失风险的

3.2 RDD的CheckPoint

将RDD数据保存起来，但是它仅支持硬盘存储。并且：

  1. 被设计认为是安全的；
  2. 不保留血缘关系；

Cache和Checkpoint的性能对比：

  • Cache性能更好，分散存储，各个Executor并行执行，效率高，保存到内存中；
  • CheckPoint较慢，集中存储，涉及到网络IO，但是存到HDFS上更安全（多副本）
