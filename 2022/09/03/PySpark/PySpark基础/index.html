Spark基础

PySpark基础

Apache Spark是一个开源的分布式集群计算框架，用于快速处理、查询和分析大数据（离线批处理和实时流式计算）。Spark执行内存计算和使用有向无环图执行引擎支持无环数据流，比Hadoop等Map Reduce框架快100倍。

Spark 2.0 版本之前, Spark 的核心编程接口是弹性分布式数据集(RDD) 。Spark2.0版本之后, RDD 被 Dataset 所取代, Dataset 跟 RDD 一样也是强类型的, 但是底层做了更多的优化。

  • RDD是一种分布式内存抽象，能够在大规模集群中做内存运算，是整个Spark的核心数据结构；

PySpark是Apache Spark的Python接口。它不仅允许使用Python api编写Spark应用程序，还提供了PySpark shell，用于在分布式环境中交互分析数据。PySpark支持Spark的大部分特性，如Spark Core、Spark SQL，DataFrame，Streaming，MLlib和GraphX（用的旧图）。

  • Spark Core：Spark Core 是 Spark 平台的底层通用执行引擎，所有其他功能都建立在其之上。它提供了RDD和内存计算能力。
  • Spark SQL and DataFrame：Spark SQL 是用于结构化数据处理的 Spark 模块。它提供了一个称为 DataFrame 的编程抽象，也可以作为分布式 SQL 查询引擎。Spark SQL针对的是离线计算场景，Spark提供了Structred Streaming模块【替代Spark Streaming】以Spark SQL为基础，进行数据流式计算。
  • Streaming：Apache Spark 中的流式处理功能在 Spark 之上运行，支持跨流式处理和历史数据的强大交互式和分析应用程序，同时继承了 Spark 的易用性和容错特性。
  • MLlib：MLlib 建立在 Spark 之上，是一个可扩展的机器学习库，它提供了一组统一的高级 API，可帮助用户创建和调整实用的机器学习pipeline。

运行模式

  1. 本地模式（单机）：以一个独立的进程，通过其内部的多个线程来模拟整个Spark运行环境。

  2. Standalone模式（集群）：Spark中的各个角色以独立进程的形式存在，并组成Spark集群环境。

  3. Hadoop YARN模式（集群）：Spark中的各个角色运行在YARN的容器内部，并组成Spark集群环境。

    运行在集群上，主要有四个部分组成：

    • Master(ResourceManager) ：集群的资源管理与分配；
    • Worker(NodeManager) ：负责管理单个服务器的资源；
    • Driver：单个Spark任务的管理者，管理Executor的任务执行和任务分解分配；
    • Executor：具体干活的进程，Spark的工作任务都是由Executor来负责执行；
