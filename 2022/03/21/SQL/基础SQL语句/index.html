基础SQL

基础SQL

一些常用的SQL关键字、操作符、语句和函数的介绍，包括了ONE SQL（Mysql SQL）、Hive SQL。

1. 关键字&语句

1.1 DISTINCT

DISTINCT关键词用于返回唯一不同的值。

【注】distinct可以看作是特殊的group by，执行效率几乎等价。

1.2 LIMIT

一般用于打印少数行来查看检索内容。

  • limit N：返回 N 条记录；
  • limit N,M：从第 N 条记录开始, 返回 M 条记录；

1.3 LIKE & REGEXP

like和regexp都是模式匹配，其中regexp支持正则表达

regex常用的正则表达有：

模式 匹配模式
^ 匹配字符串开头
$ 匹配字符串结尾
. 匹配任意字符
[…] 匹配方括号间列出的任意字符
[^…] 匹配方括号间未列出的任意字符
p1|p2|p2 交替：匹配任意p1或p2或p3
* 匹配前面的字符零次或者多次
? 匹配前面的字符零次或者1次
+ 匹配前面的字符一次或者多次
{n} 匹配前面的字符n次
{m, n} 匹配前面的字符m至n次

1.4 ORDER BY

ORDER BY关键字用于对结果集按照一个列或者多个列进行排序。

  • 若需要按多列值排序且顺序不同，可直接在列名后加上ASC|DESC

1.5 GROUP BY

GROUP BY语句用于结合聚合函数，根据一个或多个列对结果集进行分组。

1.6 HAVING

在 SQL 中增加 HAVING 子句原因是，WHERE 关键字无法与聚合函数一起使用。

HAVING 子句可以筛选分组后的各组数据。

【注】工作中从未使用过这个语句。

2. 操作符

2.1 IS NULL && IS NOT NULL

IS NULL：

IS NOT NULL：

  • 可以配合IF函数进行使用

2.2 BETWEEN

BETWEEN 操作符选取介于两个值之间的数据范围内的值。这些值可以是数值、文本或者日期。

2.3 IN

IN 操作符允许在 WHERE 子句中规定多个值。

2.4 UNION

组合查询。UNION操作符合并两个或多个 SELECT 语句的结果（组合查询），内部的每个 SELECT 语句必须拥有相同数量的列。

【注】另一个操作符是union all，区别：union all不去重，union去重。union all相对较为常用。

2.5 EXISTS

EXISTS运算符用于判断查询子句是否有记录，如果有一条或多条记录存在返回 True，否则返回 False。

【注】工作中一般用来判别表是否存在。

2.6 JOIN

  • INNER JOIN：交集，两个表必须都存在该列特征；
  • LEFT JOIN：最终查询A集合必须都存在，如果没有对应的B，该列B的值为None；
  • RIGHT JOIN：与上述相反
  • FULL JOIN：A、B的内容都必须存在，即连接后，A列特征可能有为None的，B也有可能为None的；

【注】最常用left join与inner join。

【1】JOIN…ON…中可以写若干个条件（也可以放WHERE语句）

【2】USING关键字简化连接查询，但需要满足条件：（1）等值连接；（2）必须要有相同的列

3. 函数

3.1 聚合（Aggregate）函数

SUM：返回列的总数

AVG：返回平均值

COUNT：返回匹配指定条件的行数。

MAX：返回最大值

MIN：返回最小值

3.2 统计函数

ROUND ：函数用于把数值字段舍入为指定的小数位数。

EXP：e的X方。

TRUNCATE(X, Y) ：X截断为Y位小数。

LN：自然对数。

COALESCE：可以兼顾一些特殊情况，例如值为NULL。

  • 若num为空，则返回num1，否则返回num；

【注】较为常用。

3.3 字符串函数

CONCAT：字符串合并

SUBSTRING：从字符串 s 的 start 位置截取长度为 length 的子字符串

FORMAT：函数可以将数字 x 进行格式化 “#,###.##”, 将 x 保留到小数点后 n 位，最后一位四舍五入。

REPLACE：字符串替换

SPLIT：字符串分割，返回一个数组

【注】Hive SQL。

3.4 逻辑判断

IF：IF函数也能通过判断条件来返回特定值。

  • 可以配合操作符IS NULL，来对null值的行赋值。

CASE WHEN：CASE WHEN语句能在SQL语句中织入判断逻辑。

重要‼️

3.5 日期函数

YEAR：提取年。

MONTH：提取月。

DAY：提取天。

DATE_ADD(date, INTERVAL expr type) ：向日期date，添加该类型type的时间间隔expr。type有：SECOND、MINUTE、HOUR等。

DATE_SUB(date, INTERVAL expr type) ：与上述相反。

3.6 窗口函数

窗口函数这里的“窗口”表示范围，可以理解为将原数据划分范围，即分组，然后用函数实现某些目的。

  • 专用窗口函数：RANK，DENSE_RANK，ROW_NUMBER
  • 汇总函数：MAX，MIN，COUNT，SUM，AVG

RANK：

  • rank()是排序函数，括号中不需要有参数；
  • 通过partition by将班级分类，相当于之前用过的group by子句功能，但是group by子句分类汇总会改变原数据的行数，而用窗口函数自救保持原行数；
  • 通过order by将成绩降序排列；

DENSE_RANK，ROW_NUMBER：
与rank相同：

区别：

  • 在rank()函数，如果有并列情况，会占用下一个名次的位置，比如成绩为100的学生有三个并列第一，那么99分的学生是第二名，名次是：1,1,1,4；
  • 在dense_rank()函数中，如果有并列的情况，不会占用下一个名词，同用上个例子，名次是：1,1,1,2；
  • 在row_number()函数中，会忽略并列的情况，同用上述例子，名次是：1,2,3,4;

与GROUP BY的区别：

group by子句分类汇总会改变原数据的行数，而用窗口函数自救保持原行数。
